# Stock 股票管理技能

## 功能概述

本技能提供完整的股票管理能力,包括:
- **股票信息管理**: 添加、编辑、删除股票持仓信息
- **盈亏分析**: 实时获取股票行情,计算持仓盈亏情况
- **智能提取**: 从自然语言中自动提取股票代码、名称、持仓价格等信息
- **数据持久化**: 将股票数据保存到本地 JSON 文件

## 可用工具

### 1. getStockTableWithProfit
**功能**: 以 Markdown 表格形式展示所有股票的持仓及盈亏情况

**返回格式**:
```
| 股票名称 | 持仓价格 | 持仓数量 | 当前价格 | 浮盈 |
|---------|---------|---------|---------|------|
| xxx    | xx.xx   | xx.xx   | xx.xx   | +xx.xx |
```

**字段说明**:
- **持仓价格**: 每股成本价格(不是总成本)
- **当前价格**: 实时市场价格
- **浮盈**: (当前价格 - 持仓价格) × 持仓数量,正数表示盈利(+),负数表示亏损(-)

**使用场景**: 用户查询持仓情况、查看盈亏状态时调用

### 2. writeJson
**功能**: 新增或编辑股票信息到本地 JSON 文件

**参数**:
```json
{
  "stockCode": "股票代码",
  "name": "股票名称",
  "stockType": "股票类型(默认A股)",
  "holdingPrice": "持仓价格",
  "holdingQuantity": "持仓数量"
}
```

**返回值**: `true` 表示成功,`false` 表示失败

**使用场景**: 
- 用户添加新股票持仓
- 用户修改现有股票信息(如调整持仓价格或数量)

### 3. deleteJson
**功能**: 从本地 JSON 文件中删除指定股票

**参数**: `stockCode` (股票代码字符串)

**返回值**: `true` 表示成功,`false` 表示失败

**使用场景**: 用户删除不再持有的股票

## 工作流程

### 场景1: 查询持仓盈亏
1. 用户询问持仓情况或盈亏状态
2. 调用 `getStockTableWithProfit()` 工具
3. 在返回结果外层包裹 `<pre>` 标签后展示给用户
4. 可选:对盈亏情况进行简要分析说明

### 场景2: 添加/修改股票
1. 从用户输入中提取股票信息(使用 `extractStockFrom` 方法)
   - 默认 `stockType` 为 "A"
2. 调用 `writeJson(stock)` 工具保存数据
3. 根据返回值回复:
   - 成功: "**保存成功** ✅"
   - 失败: "**保存失败** ❌"

### 场景3: 删除股票
1. 从用户输入中提取股票代码
2. 调用 `deleteJson(stockCode)` 工具删除数据
3. 根据返回值回复:
   - 成功: "**删除成功** ✅"
   - 失败: "**删除失败** ❌"

## 注意事项

1. **股票类型默认值**: 除非用户明确指定,否则 `stockType` 默认为 "A" (A股)
2. **价格精度**: 所有价格保留两位小数显示
3. **浮盈计算**: 浮盈 = (当前价格 - 持仓价格) × 持仓数量,需明确区分单价和总成本
4. **数据源**: 股票数据存储在本地 JSON 文件,路径由配置项 `app.json.storage.path` 指定
5. **实时行情**: 盈亏计算依赖实时行情数据,通过 `StockApiService` 获取
6. **Markdown 格式**: 展示表格时务必在结果外层添加 `<pre></pre>` 标签以保持格式

1. 提取股票信息: `{stockCode: "600519", name: "贵州茅台", holdingPrice: 1800, holdingQuantity: 100, stockType: "A"}`

3. 回复: "**保存成功** ✅"

**用户**: "把平安银行从持仓里删掉"
**助手**: 
1. 识别股票代码: "000001"
2. 调用 `deleteJson("000001")`
3. 回复: "**删除成功** ✅"
